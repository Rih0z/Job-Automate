# make-slide.md スライド 品質レビュープロンプト

> **対象**: `make-slide.md` プロンプトをもとに作成された HTML/CSS プレゼンテーションスライド
> **目的**: 「技術準拠・HTML/CSS品質」「視認性・コントラスト」「情報量・レイアウト適切性」「コンテンツ構成・論理性」「プレゼン効果・伝達力」の5軸で品質を100点満点で評価する。

---

## レビュー前の必須確認事項

レビューを開始する前に、以下をスライドまたは作成者から確認すること。

| 確認項目 | 確認内容 |
|----------|----------|
| **このプレゼンで最も伝えたいことは何か？** | 主張・結論を1〜2文で言語化できるか |
| **想定聴衆は誰か？** | 技術者・非技術者・経営層など、どの層を主対象にしているか |
| **聴衆がこのスライドを見た後に何を「できる」「わかる」ようになるか？** | 行動変容または理解のゴールが明示されているか |
| **make-slide.md のどのバージョンを適用したか？** | v4.1等、特殊な省略・変更があれば事前に把握する |

上記が不明瞭な場合は、レビュー開始前に作成者へ確認すること。

## レビュー観点（5項目・各20点）

---

### 1. make-slide.md 技術準拠・HTML/CSS品質 【配点：20点】

> **目的：** `make-slide.md` の実装ルールが正しく適用されており、スライドの HTML/CSS および JavaScript が正しく機能するかを確認する。

#### 1-A. CSS技術ルール遵守チェック

- [ ] **CSS変数（`var(--xxx)`）が一切使用されていない**（直接カラーコード指定）
- [ ] **すべてのテキスト要素に明示的な `color` が指定されている**
- [ ] **`color: white` 等の曖昧な記述がなく、`#ffffff` 等のカラーコードを使用している**
- [ ] **スクロールバーが有効化されている**（`.slide-content` に `overflow-y: auto`）
- [ ] **`box-sizing: border-box`** が全要素に適用されている

#### 1-B. スライド構造チェック

- [ ] **タイトルスライド**（`.title-slide`）が先頭にある
- [ ] **セクション区切り**（`.section-divider`）が適切な箇所に配置されている
- [ ] **最終スライド**がサマリー形式（`.summary-slide`）になっている
- [ ] スライド番号表示（`#slideNumber`）が実装されている
- [ ] プログレスバー（`#progressBar`）が実装されている
- [ ] PDF出力ボタン（`.pdf-export-btn`）が実装されている
- [ ] ナビゲーションエリア（`.nav-prev` / `.nav-next`）が実装されている
- [ ] ヘルプオーバーレイ（`#helpOverlay`）が実装されている

#### 1-C. JavaScript機能チェック

- [ ] **キーボードナビゲーション**（← → Space Backspace Home End F ?）が実装されている
- [ ] **タッチスワイプ**によるスライド移動が実装されている
- [ ] **PDF出力時の確認ダイアログ**（横向き・背景グラフィックON等の案内）が表示される
- [ ] スライド切り替え時に `scrollTop` がリセットされる
- [ ] `PresentationManager` クラスが正しく初期化されている

#### 1-D. 印刷・PDF出力スタイルチェック

- [ ] `@media print` スタイルが実装されている
- [ ] 印刷時に全スライドが表示される（`display: block !important`）
- [ ] 印刷時にナビゲーション・プログレスバー・PDFボタン等が非表示になる
- [ ] 印刷時に背景色・グラデーションが保持される（`-webkit-print-color-adjust: exact`）
- [ ] 印刷時にアニメーション要素が可視化される（`opacity: 1 !important`）
- [ ] 印刷用フォントサイズが適切に設定されている（h1: 28px、h2: 22px、p/li: 16px）

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 全技術ルールが遵守されており、全機能が正常に動作する |
| 14〜17 | ほぼ遵守されているが、軽微な技術的問題がある（一部 CSS 変数使用・スクロール不備等） |
| 10〜13 | CSS 変数使用や機能の欠落など、複数の技術的問題がある |
| 0〜9  | 基本的な実装ルールが守られておらず、機能不全が多数ある |

### 2. 視認性・コントラスト 【配点：20点】

> **目的：** `make-slide.md` のフォントサイズガイドラインおよびコントラスト規定が遵守されており、遠くから（プロジェクター等）でも読みやすいスライドになっているかを確認する。

#### フォントサイズチェック（各スライドで確認）

- [ ] スライドタイトル（h1）が **36px 以上（最小 32px）**
- [ ] セクション見出し（h2）が **26px 以上（最小 24px）**
- [ ] 小見出し（h3）が **20px 以上（最小 18px）**
- [ ] 本文・箇条書き（p, li）が **20px 以上（最小 18px）**
- [ ] 補足・注釈が **16px 以上（最小 14px）**
- [ ] タイトルスライドの h1 が **48px**（専用定義）
- [ ] フォントを小さくしてコンテンツを無理に収めていないか（小さくするのではなく分割する）

#### コントラストチェック（各背景色別に確認）

- [ ] **タイトルスライド（濃い背景 #003366）の文字が白（#ffffff）**
- [ ] **タイトルスライドの `.subtitle` が #e0e0e0、`.date` が #aaaaaa 以上の明度**
- [ ] **通常スライド（白背景 #ffffff）の本文が濃いグレー（#333333）**
- [ ] **カード内テキスト（h3: #0066cc, p: #333333）が背景 #f8f8f8 に対して読める**
- [ ] **CTAボックス内のすべてのテキストが白（#ffffff）**
- [ ] **テーブルのヘッダー（濃紺背景 #003366）が白文字（#ffffff）**
- [ ] **セクション区切りスライド（グレー背景）の見出しが濃紺（#003366）**
- [ ] **`strong` 要素の色（#0066cc）が周囲の背景に対して読める**

#### 全体的な視認性

- [ ] 遠くからでも各スライドの主要情報が視認できる
- [ ] セクション区切りスライドの見出しが大きく、明確に識別できる
- [ ] 日本語フォント（游ゴシック・Noto Sans JP 等）が適切に指定されている

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 全フォントサイズが規定値以上、全コントラストが確保されており、視認性に問題がない |
| 14〜17 | ほぼ適切だが、一部でフォントサイズ不足またはコントラスト不足がある |
| 10〜13 | フォントサイズ違反またはコントラスト問題が複数あり、読みにくいスライドが存在する |
| 0〜9  | 視認性に深刻な問題があり、プレゼンツールとして機能しない |

### 3. 情報量・レイアウト適切性 【配点：20点】

> **目的：** `make-slide.md` の「1スライドあたりの情報量ガイドライン」が遵守されており、コンテンツがはみ出さず適切なレイアウトで表示されているかを確認する。

#### 情報量チェック（全スライドで確認）

- [ ] 箇条書き（トップレベル）が **5項目以内**
- [ ] 箇条書き（ネスト含む合計）が **10項目以内**
- [ ] テーブルの行数が **6行以内**
- [ ] カードの数が **4枚以内**
- [ ] 段落テキストが **3段落・計150文字以内**
- [ ] 2カラムの各カラムにカードが **3枚未満**
- [ ] フォントサイズを小さくして情報を詰め込んでいない（情報を減らすかスライドを分割する）

#### スライド分割判断チェック

以下の条件に該当するスライドが適切に分割されているかを確認：

- [ ] 箇条書きが6項目以上あるスライドが分割されている
- [ ] テーブルが7行以上あるスライドが分割されている
- [ ] テキストが200文字以上あるスライドが分割されている
- [ ] 2カラムの両方に3枚以上のカードがあるスライドが分割されている

#### レイアウトチェック

- [ ] コンテンツがスライド外にはみ出していない（overflow対策が機能している）
- [ ] 情報量に応じた適切なレイアウトが選択されている（1カラム・2カラム・3カラム）
- [ ] スライドの余白（padding）が適切で、窮屈な印象がない
- [ ] カードのレイアウトが崩れていない
- [ ] テーブルがスライド幅に収まっている
- [ ] ロードマップ（タイムライン）が正しく表示されている（使用している場合）
- [ ] 参加者リスト（`.participant-grid`）が正しく表示されている（使用している場合）

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 全スライドが情報量ガイドライン内に収まり、レイアウトが崩れていない |
| 14〜17 | ほぼ適切だが、1〜2枚のスライドで情報過多またはレイアウトの軽微な崩れがある |
| 10〜13 | 複数のスライドで情報量超過またはレイアウト崩れがある |
| 0〜9  | 多数のスライドで深刻なはみ出し・崩れがあり、プレゼンとして使用不可 |

### 4. コンテンツ構成・論理性 【配点：20点】

> **目的：** スライド全体の流れが論理的であり、聴衆が内容を追いやすい構成になっているかを確認する。

#### 構成チェック

- [ ] **タイトルスライド**にタイトル・サブタイトル・発表者・日付が含まれている
- [ ] **目次またはアジェンダスライド**がある（必要な場合）
- [ ] **セクション区切りスライド**がセクション移行時に使用されている
- [ ] スライドの流れが論理的な順序（課題→解決策→根拠→まとめ、等）になっている
- [ ] 各スライドに **1つの明確なテーマ・メッセージ** がある
- [ ] スライド間の **文脈的なつながり** が明確（前後関係が分かる）

#### 最終スライドチェック（必須）

- [ ] 最終スライドが **「Thank You」「ご清聴ありがとうございました」だけで終わっていない**
- [ ] 最終スライドが **サマリー形式**（`.summary-slide`）になっている
- [ ] **重要ポイントが3〜5個**まとめられている
- [ ] 各ポイントに **参照ページ番号**（→ p.X 形式の `.summary-page`）が付いている
- [ ] **Next Action / 次のステップ**が記載されている（必要な場合）

#### 視覚要素の適切性

- [ ] 強調すべき内容に `.key-point` や `.highlight` が使われている
- [ ] CTA（行動促進）がある場合、`.cta-box` で適切に強調されている
- [ ] データ・統計はテーブルや視覚的に整理された形式で提示されている
- [ ] ロードマップ・工程表がある場合、`.roadmap` スタイルが使用されている

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 論理的な構成で、最終スライドがサマリー形式になっており、各スライドのテーマが明確 |
| 14〜17 | おおむね良好だが、最終スライドの不備または一部スライドの構成に問題がある |
| 10〜13 | 最終スライドが「Thank You」のみ、または複数スライドで構成が混乱している |
| 0〜9  | 論理的な流れがなく、聴衆が内容を追うことが困難 |

---

### 5. プレゼン効果・伝達力 【配点：20点】

> **目的：** スライドが聴衆への効果的な伝達ツールとして機能しているかを確認する。1スライド1メッセージの原則が守られ、視覚的要素が内容の理解を助けているかを評価する。

- [ ] 各スライドの **タイトル1行で何を伝えたいかがわかる**（タイトル自体がメッセージになっている）
- [ ] 箇条書きが **体言止めまたは短い文** で統一されている（長文の羅列になっていない）
- [ ] **カラーバリエーション**（`.card.accent` / `.card.success`）が情報の重要度・種類に合わせて使われている
- [ ] アニメーション（`.animate-item`）が過剰でなく、情報の提示順序に意味がある（使用している場合）
- [ ] スライドのデザイン・配色が **プレゼン全体を通じて一貫している**
- [ ] 聴衆が「自分事」として受け取れる **具体例・事例** がある
- [ ] 数字・データが **視覚的に強調** されている（大きなフォント・カラー等）
- [ ] プレゼンの流れが **起承転結または問題提起→解決策→効果** の構造になっている
- [ ] スライドに **不要な装飾や冗長な情報** がなく、シンプルに整理されている
- [ ] 質疑応答を促すような **問いかけや議論ポイント** がある（必要な場合）

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 1スライド1メッセージが徹底されており、視覚的要素が効果的に伝達力を高めている |
| 14〜17 | おおむね効果的だが、一部で情報が整理されていないスライドがある |
| 10〜13 | 伝達力が弱く、複数スライドで複数のメッセージが混在している |
| 0〜9  | スライド全体を通じて伝達力が不足しており、聴衆に伝わるか懸念される |

## 重要な注意事項

> **レビュワーは元ファイルを絶対に編集してはならない**
>
> レビューはあくまで「フィードバックの提供」であり、修正作業は作成者自身が行う。
> 改善点は具体的に指摘し、修正案は「提案」として記載すること。
>
> CSS技術的な問題（CSS変数使用・コントラスト不足・フォントサイズ違反等）については、
> make-slide.md の該当バージョンの絶対ルール番号を明記すること（例：v4.1 絶対ルール#1「CSS変数禁止」）。

## 使用方法

1. このプロンプトと一緒に、レビュー対象の HTML スライドファイルを添付する
2. 「このスライドをレビューしてください」と依頼する
3. 「レビュー前の必須確認事項」が文書から読み取れない場合は、作成者へ確認する
4. 採点結果を `.tmp/` ディレクトリに `review-<スライド名>-<YYYYMMDD>.md` 形式で格納する
5. B判定以下の場合は、具体的な修正箇所リストを作成者に共有し、再レビューを実施する
