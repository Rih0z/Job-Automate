# リサーチ・情報収集 品質レビュープロンプト

> **対象**: `business/news/*.md` および `business/seo/*.md` プロンプトで生成されたリサーチ・ニュース収集・SEO分析の成果物
> **目的**: 「情報の正確性・信頼性」「網羅性・カバレッジ」「分析の深度・洞察」「構成・可読性」「実用性・アクション接続」の5軸でリサーチ成果物を100点満点で評価する。

---

## レビュー前の必須確認事項

レビューを開始する前に、以下を成果物または作成者から確認すること。

| 確認項目 | 確認内容 |
|----------|----------|
| **どのプロンプトで生成された成果物か？** | `business/news/*.md`（ニュース収集系）か `business/seo/*.md`（SEO分析系）かを特定する |
| **対象とする地域・市場はどこか？** | シアトル・全米・日本など、リサーチ対象の地域が明確か |
| **対象期間はいつか？** | 直近24〜72時間か、特定の期間分析か |
| **この成果物の利用目的は何か？** | 投資判断・事業戦略・コンテンツ企画・市場参入検討など、用途を確認する |
| **想定読者は誰か？** | 経営層・マーケター・エンジニア・投資家など、読者に応じてレビュー深度を調整する |

上記が不明瞭な場合は、レビュー開始前に作成者へ確認すること。

---

## レビュー観点（5項目・各20点・合計100点）

---

### 1. 情報の正確性・信頼性 【配点：20点】

> **目的：** 収集された情報が正確であり、信頼できるソースに基づいているかを確認する。事実と推測の区別、データの鮮度、バイアスの有無を評価する。

- [ ] 引用されたソースが**主要メディア・専門メディア・公的機関**など信頼性の高い情報源であるか
- [ ] 各情報に**引用元URL**が明記されており、アクセス可能か
- [ ] **事実**と**推測・見解**が明確に区別して記述されているか
- [ ] 掲載データの**日付・時点**が明記されており、情報の鮮度が確認できるか
- [ ] **数値データ**（市場規模・成長率・検索ボリューム等）に出典が付されているか
- [ ] SNSのみを一次ソースとする情報が排除されているか（裏付けなしのSNS情報の混入がないか）
- [ ] 特定企業・製品・立場に偏った**バイアス**が含まれていないか
- [ ] フェイクニュース・未確認情報が排除されているか
- [ ] 課金登録が必要なソースに偏っていないか（アクセシビリティの確保）

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | すべてのソースが信頼性の高いメディア・機関であり、引用元が完備。事実と推測が明確に区別され、バイアスが排除されている |
| 14〜17 | おおむね信頼性の高いソースだが、一部の引用元が欠落または信頼性が低い。事実と推測の区別が一部曖昧 |
| 10〜13 | ソースの信頼性にばらつきがあり、引用元の欠落が目立つ。バイアスの可能性がある記述が含まれている |
| 0〜9  | ソースの信頼性が低い、または引用元がほぼ示されていない。事実と推測が混在し、情報の正確性に疑問がある |

---

### 2. 網羅性・カバレッジ 【配点：20点】

> **目的：** リサーチ対象に対して必要な情報が漏れなく収集されているか、複数の視点を包含しているかを確認する。

- [ ] プロンプトで指定された**主要トピック・カテゴリ**がすべてカバーされているか
- [ ] 指定された**地域（シアトル・全米・日本等）**すべてについて情報が収集されているか
- [ ] 同一トピックに対して**賛否・異なる立場**からの視点が含まれているか
- [ ] 関連する**トレンド・背景情報**との接続が示されているか
- [ ] **グローバル視点**と**ローカル視点**の両方が提供されているか
- [ ] 該当ニュースが存在しないカテゴリについて「**該当なし**」と明記されているか（無言の欠落がないか）
- [ ] SEO分析の場合：**検索意図・競合・ユーザーペインポイント**の3要素が揃っているか
- [ ] ニュース収集の場合：**事実・影響・今後の見通し**の3要素が揃っているか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 指定されたすべてのカテゴリ・地域がカバーされ、複数視点が提供されている。関連トレンドとの接続も示されている |
| 14〜17 | 主要なカテゴリ・地域はカバーされているが、一部の視点やトレンド接続が不足している |
| 10〜13 | カバレッジに明確な抜け漏れがあり、特定のカテゴリや地域が欠落している |
| 0〜9  | カバレッジが大幅に不足しており、指定された範囲の半分以上が欠落している |

---

### 3. 分析の深度・洞察 【配点：20点】

> **目的：** 単なる情報の羅列に留まらず、「So What?（だから何？）」の洞察が含まれているかを確認する。因果関係の分析、将来予測、アクション可能な示唆の有無を評価する。

- [ ] 個別のニュース・データに対して**「だから何？」（So What?）**の洞察が添えられているか
- [ ] 複数の情報を横断した**因果関係の分析**が含まれているか
- [ ] **将来予測・トレンド予想**が示されており、その根拠が明記されているか
- [ ] 表面的な要約ではなく、**市場影響・業界へのインパクト**が分析されているか
- [ ] SEO分析の場合：**競合の弱点を突く具体的な戦略**が提案されているか
- [ ] ニュース収集の場合：**各ニュースの相互関連性**（点と点を線で結ぶ分析）が示されているか
- [ ] データの**傾向・パターン**が抽出されているか
- [ ] 短期的影響と**中長期的影響**の両方が考察されているか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 情報の羅列を超えた深い洞察があり、因果関係の分析・将来予測・相互関連性の分析が充実している。読み手に新たな気づきを与えるレベル |
| 14〜17 | 一定の洞察はあるが、因果関係の分析や将来予測の根拠がやや薄い。一部は表面的な要約に留まっている |
| 10〜13 | 洞察が断片的で、情報の羅列に近い。「So What?」の視点が大部分で欠けている |
| 0〜9  | 洞察がほぼなく、ニュースや情報の転記・要約のみ。分析と呼べる内容がない |

---

### 4. 構成・可読性 【配点：20点】

> **目的：** 情報の整理・分類が適切で、読者が短時間で要点を把握できる構成になっているかを確認する。

- [ ] 情報が**カテゴリ別・地域別**に適切に整理・分類されているか
- [ ] **重要度順**（または時系列順）に情報が並べられており、優先順位が明確か
- [ ] **見出し・表・箇条書き**が効果的に活用されており、視認性が高いか
- [ ] プロンプトで指定された**出力フォーマット**（Markdown表・指定カラム等）に準拠しているか
- [ ] 要約が簡潔で、**1項目あたりの文字数制限**（70字以内等）が守られているか
- [ ] **表記の統一性**が保たれているか（日付形式・カテゴリ名・用語の統一）
- [ ] 冗長な記述がなく、**情報密度が適切**か（短すぎず長すぎず）
- [ ] 読者が**3分以内で全体像を把握**できる構成になっているか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 指定フォーマットに完全準拠し、情報が論理的に整理されている。見出し・表・箇条書きの活用が効果的で、読者が短時間で要点を把握できる |
| 14〜17 | おおむねフォーマットに準拠しているが、一部の分類や並び順に改善余地がある。可読性は確保されている |
| 10〜13 | フォーマットの逸脱が目立ち、情報の整理が不十分。読者が要点を把握するのに時間がかかる |
| 0〜9  | 構成が混乱しており、情報の分類・整理がされていない。フォーマット指定がほぼ無視されている |

---

### 5. 実用性・アクション接続 【配点：20点】

> **目的：** リサーチ成果物が読み手のビジネス判断に活用でき、次のアクション（投資判断・施策検討・コンテンツ制作等）へ具体的に接続されているかを確認する。

- [ ] 収集された情報が読み手の**ビジネス判断・意思決定に直接活用**できるか
- [ ] **次のアクション**（投資判断・施策検討・コンテンツ制作・市場参入検討等）への接続が示されているか
- [ ] **競合情報・市場機会**が具体的に特定されているか
- [ ] SEO分析の場合：**キーワード戦略が実行可能**であり、具体的な記事構成・導線設計まで落とし込まれているか
- [ ] SEO分析の場合：**マネタイズ戦略**（CTA配置・成約率向上策）が実装可能なレベルで記載されているか
- [ ] ニュース収集の場合：各ニュースが**自社ビジネスへの影響・対応策**として解釈されているか
- [ ] 情報の**優先順位づけ**がされており、「まず何をすべきか」が明確か
- [ ] 成果物を**そのまま社内共有・報告**に使えるクオリティか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | リサーチ結果が具体的なアクションに直結しており、ビジネス判断に即活用できる。競合情報・市場機会の特定が明確で、次のステップが示されている |
| 14〜17 | 実用性はあるが、アクションへの接続が一部抽象的。競合情報や市場機会の特定にもう一段の具体性が欲しい |
| 10〜13 | 情報としての価値はあるが、ビジネス判断への活用方法が不明確。アクション接続がほぼない |
| 0〜9  | 情報の羅列のみで、実用性・アクション接続が皆無。読み手が自力で分析し直す必要がある |

---

## 採点方法

### 総合点：100点満点

| 評価 | 点数 | 判定 |
|------|------|------|
| S | 90〜100点 | そのままビジネス判断・社内共有に活用できる品質。追加調査不要 |
| A | 80〜89点 | 軽微な補足・修正後に活用可能。特定の観点で追加情報を補えばよい |
| B | 70〜79点 | 部分的に活用可能だが、特定の観点で再調査が必要 |
| C | 60〜69点 | 大幅な再調査・再分析が必要。構成や分析の抜本的な見直しが求められる |
| D | 59点以下 | 成果物の品質が基準を大きく下回り、プロンプトの再設計または手動リサーチが必要 |

## 重要な注意事項

> **レビュワーは元ファイルを絶対に編集してはならない**
>
> レビューはあくまで「フィードバックの提供」であり、修正作業は作成者自身が行う。
> 改善点は具体的に指摘し、修正案は「提案」として記載すること。

> **プロンプト種別に応じた評価調整**
>
> ニュース収集系（`business/news/*.md`）とSEO分析系（`business/seo/*.md`）では重視すべき観点が異なる。
> - ニュース収集系：「情報の正確性・信頼性」と「網羅性・カバレッジ」を特に重視する
> - SEO分析系：「分析の深度・洞察」と「実用性・アクション接続」を特に重視する
>
> 評価コメントにはこの観点の重み付けを明記すること。

---

## 使用方法

1. このプロンプトと一緒に、レビュー対象のリサーチ成果物を貼り付ける
2. 「このリサーチ成果物をレビューしてください」と依頼する
3. S/A判定であればそのままビジネス判断・社内共有に活用する
4. B以下の場合は改善提案を反映し、該当プロンプトで再生成または手動で情報を補完する
