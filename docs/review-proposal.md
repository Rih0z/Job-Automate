# 提案書 品質レビュープロンプト

> **対象**: `business-proposal.md` / `it-proposal.md` / `generic-proposal.md` をもとに作成された提案書・企画書
> **目的**: 「構成・論理性」「市場分析・実現可能性」「ペルソナ適合性」「根拠の明確性」「説得力・表現品質」の5軸で提案書品質を100点満点で評価する。

---

## レビュー前の必須確認事項

レビューを開始する前に、以下を提案書または作成者から確認すること。

| 確認項目 | 確認内容 |
|----------|----------|
| **この提案書で最も伝えたいことは何か？** | 主張・結論を1〜2文で言語化できるか |
| **提案先（読者・意思決定者）は誰か？** | 経営層・担当者・投資家など、どの層を主対象にしているか |
| **読者がこの提案書を読んだ後に何を「決断する」か？** | 求める行動（承認・投資・採用など）が明示されているか |
| **どのプロンプトで作成したか？** | business-proposal / it-proposal / generic-proposal のどれか |

上記が不明瞭な場合は、レビュー開始前に作成者へ確認すること。

---

## レビュー観点（5項目・各20点）

---

### 1. 構成・論理性 【配点：20点】

> **目的：** 提案書全体の流れが論理的であり、読者が内容を追いやすい構成になっているかを確認する。

- [ ] **エグゼクティブサマリー**（要約・結論）が冒頭にある
- [ ] **課題の明示** → **解決策の提示** → **根拠・実績** → **実装計画** → **期待効果** の順序で展開されている
- [ ] 各セクションに**明確な見出し**があり、読者がどこを読んでいるか把握できる
- [ ] 結論が冒頭と末尾の両方に記載されている（BLOT原則：Bottom Line On Top）
- [ ] セクション間の**文脈的なつながり**が明確（前後関係が分かる）
- [ ] 提案の規模・リスク・期待値が**定量的に**示されている

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 論理的な構成で、結論が冒頭から明確、各セクションが有機的につながっている |
| 14〜17 | おおむね良好だが、一部セクションの順序または文脈的つながりに問題がある |
| 10〜13 | 構成が不明確で、読者が結論や主張を把握しにくい |
| 0〜9  | 論理的な流れがなく、提案書として機能しない |

---

### 2. 市場分析・実現可能性 【配点：20点】

> **目的：** TAM/SAM/SOM分析が含まれ、事業・提案の実現可能性が客観的根拠とともに示されているかを確認する。

#### 市場規模分析チェック

- [ ] **TAM**（Total Addressable Market：全体市場規模）が数値で示されている
- [ ] **SAM**（Serviceable Addressable Market：実際にアプローチできる市場）が定義されている
- [ ] **SOM**（Serviceable Obtainable Market：現実的に獲得できる市場）が示されている
- [ ] 市場規模の根拠（データソース・調査結果）が明記されている

#### 実現可能性チェック

- [ ] 実装に必要なリソース（人員・予算・期間）が具体的に示されている
- [ ] 主要なリスクと対策が記載されている
- [ ] 競合・代替手段との比較が含まれている
- [ ] 段階的な実装計画（フェーズ分け）がある

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | TAM/SAM/SOM分析が揃い、リソース・リスク・競合比較がすべて含まれている |
| 14〜17 | 市場分析は含まれるが、TAM/SAM/SOMの一部が欠けるか根拠が不十分 |
| 10〜13 | 市場分析が表面的で、実現可能性の根拠が乏しい |
| 0〜9  | 市場分析・実現可能性の記述がほぼなく、信頼性に問題がある |

---

### 3. ペルソナ・読者適合性 【配点：20点】

> **目的：** 提案書の内容・語彙・説明レベルが、想定する意思決定者・読者に最適化されているかを確認する。

- [ ] **ペルソナ分析**（意思決定者の課題・ニーズ・懸念点）が含まれているか
- [ ] 提案の内容が、ペルソナの**業務上の痛み**に直接応えているか
- [ ] 専門用語に対して**適切な解説**が付いているか（読者リテラシーに合わせた説明）
- [ ] 読者が「**自分事**」として捉えられるような具体例・事例があるか
- [ ] 意思決定者が懸念しそうな**反論への先回り**（Q&A・FAQ等）があるか
- [ ] 提案の**緊急性・重要性**が読者の視点で語られているか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | ペルソナ分析に基づき、読者の課題・懸念点に的確に応えている |
| 14〜17 | おおむね適切だが、一部でペルソナとのズレまたは反論への対応が不足 |
| 10〜13 | 読者への配慮が不足しており、誰向けか不明確な部分がある |
| 0〜9  | ペルソナへの適合がほぼなく、一方的な情報提供になっている |

---

### 4. 根拠の明確性 【配点：20点】

> **目的：** 主張・数値・結論が客観的な根拠に基づいており、読者が独立して評価・検証できる記述になっているかを確認する。

- [ ] 主張・推奨・結論に明確な根拠（データ・実績・文献・調査）が示されているか
- [ ] 数値・統計・効果予測に出典または測定方法が明記されているか
- [ ] 「〜と考えられる」「〜を見込む」など、確証のない推測に適切な言及があるか
- [ ] 事実として断定している記述と、著者の見解・推測が区別されているか
- [ ] 反論・リスク・制約条件が公平に取り上げられているか
- [ ] 読者が「自分の環境でも試せる」または「独自に検証できる」情報があるか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 根拠・反論・限界が明示され、読者が独立して判断できる |
| 14〜17 | おおむね合理的だが、一部で根拠不足または誘導的な記述がある |
| 10〜13 | 根拠が薄く、主観的な主張が目立つ |
| 0〜9  | 根拠がほぼなく、客観的評価が困難 |

---

### 5. 説得力・表現品質 【配点：20点】

> **目的：** 提案書が読者を動かす説得力を持ち、文章・構成・ビジュアル要素が高品質かを確認する。

- [ ] タイトル・見出しが提案の本質を一言で表現しているか
- [ ] **CTA（行動促進）** が明確に記載されている（「〜を承認してください」「〜を採用してください」等）
- [ ] 数字・データが視覚的に強調されている（表・グラフ・太字等）
- [ ] 文章が簡潔で、冗長な記述や不要な修飾語がないか
- [ ] 提案全体を通じて**一貫したトーン・スタイル**が維持されているか
- [ ] 読者が読了後に「**次のアクション**」を明確にイメージできるか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | 説得力が高く、CTAが明確、全体を通じて品質が一貫している |
| 14〜17 | おおむね説得力があるが、一部でCTAが弱いまたは表現の冗長さがある |
| 10〜13 | 説得力が弱く、読者が次のアクションをイメージしにくい |
| 0〜9  | 説得力が不足しており、提案書として機能しない |

---

## 採点方法

### 総合点：100点満点

| 評価 | 点数 | 判定 |
|------|------|------|
| S | 90〜100点 | そのまま提出可 |
| A | 80〜89点 | 軽微な修正後に提出可 |
| B | 70〜79点 | 修正推奨（フィードバック反映が必要） |
| C | 60〜69点 | 大幅な修正が必要 |
| D | 59点以下 | 構成・方針から見直しが必要 |

## 重要な注意事項

> **レビュワーは元ファイルを絶対に編集してはならない**
>
> レビューはあくまで「フィードバックの提供」であり、修正作業は作成者自身が行う。
> 改善点は具体的に指摘し、修正案は「提案」として記載すること。

---

## 使用方法

1. このプロンプトと一緒に、レビュー対象の提案書（マークダウンまたはテキスト）を貼り付ける
2. 「この提案書をレビューしてください」と依頼する
3. 「レビュー前の必須確認事項」が文書から読み取れない場合は、作成者へ確認する
4. B判定以下の場合は、具体的な修正箇所リストを作成者に共有し、再レビューを実施する
