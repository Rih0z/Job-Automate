# ビジネスアイデア 品質レビュープロンプト

> **対象**: `business-idea.md` プロンプトをもとに発展・整理されたビジネスアイデア文書
> **目的**: 「独自性・差別化」「市場性・成長性」「実現可能性」「ペルソナ明確性」「展開計画の具体性」の5軸でビジネスアイデアを100点満点で評価する。

---

## レビュー前の必須確認事項

レビューを開始する前に、以下をアイデア文書または作成者から確認すること。

| 確認項目 | 確認内容 |
|----------|----------|
| **このビジネスアイデアで解決したい課題は何か？** | 課題を1〜2文で言語化できるか |
| **誰のために・誰が使うビジネスか？** | ターゲット顧客・ペルソナが明示されているか |
| **このアイデアをどの段階まで発展させたいか？** | 検討段階・投資検討・実行フェーズのどれか |
| **どのプロンプト（business-idea.md）のステップを使ったか？** | 使用ステップに応じてレビュー深度を調整する |

上記が不明瞭な場合は、レビュー開始前に作成者へ確認すること。

---

## レビュー観点（5項目・各20点）

---

### 1. 独自性・差別化 【配点：20点】

> **目的：** このビジネスアイデアが既存の競合・代替手段と明確に差別化されており、独自の価値提案（UVP）を持っているかを確認する。

- [ ] **独自の価値提案（UVP: Unique Value Proposition）**が1〜2文で明確に言語化されているか
- [ ] 既存の競合サービス・代替手段との比較が含まれているか
- [ ] 競合が提供していない「このアイデアだけができること」が示されているか
- [ ] 差別化要因に持続性があるか（技術・特許・ネットワーク効果・ブランド等による参入障壁）
- [ ] 「なぜ今このビジネスなのか」という時代背景・市場タイミングの説明があるか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | UVPが明確で競合比較が揃い、差別化の持続性と市場タイミングが示されている |
| 14〜17 | おおむね独自性はあるが、競合比較または差別化の持続性の説明が不十分 |
| 10〜13 | 独自性の説明が表面的で、既存サービスとの違いが不明確 |
| 0〜9  | 既存ビジネスとほぼ同一で、差別化要因がほぼ示されていない |

---

### 2. 市場性・成長性 【配点：20点】

> **目的：** TAM/SAM/SOM分析に基づき、このビジネスアイデアが十分な市場規模と成長ポテンシャルを持っているかを確認する。

- [ ] **TAM**（Total Addressable Market：全体市場規模）が数値で示されている
- [ ] **SAM**（Serviceable Addressable Market：アプローチ可能市場）が定義されている
- [ ] **SOM**（Serviceable Obtainable Market：獲得可能市場）と初期目標が示されている
- [ ] 市場の成長トレンド（現在の成長率・将来予測）が含まれているか
- [ ] 市場規模の根拠（データソース・調査結果）が明記されているか
- [ ] **マネタイズ方法**（収益モデル・価格設定）が具体的に示されているか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | TAM/SAM/SOM分析が揃い、成長トレンドと収益モデルが具体的に示されている |
| 14〜17 | 市場分析は含まれるが、TAM/SAM/SOMの一部または収益モデルが不十分 |
| 10〜13 | 市場分析が表面的で、規模感や成長性の根拠が乏しい |
| 0〜9  | 市場分析がほぼなく、ビジネスとして成立するかの評価が困難 |

---

### 3. 実現可能性 【配点：20点】

> **目的：** このビジネスアイデアを実際に立ち上げるために必要なリソース・リスクが現実的に評価されているかを確認する。

#### リソース評価チェック

- [ ] 立ち上げに必要な**初期投資・コスト**が概算で示されているか
- [ ] 必要な**人材・スキル・技術**が特定されているか
- [ ] **MVP（Minimum Viable Product：最小限の製品）**または検証可能な最初のステップが示されているか

#### リスク評価チェック

- [ ] **主要リスク**（市場リスク・技術リスク・規制リスク・競合リスク等）が列挙されているか
- [ ] 各リスクに対する**緩和策・対応策**が示されているか
- [ ] 「うまくいかない場合」の**ピボット（軌道修正）の可能性**に言及があるか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | リソース・MVP・リスクと対策がすべて含まれ、現実的な評価がされている |
| 14〜17 | おおむね評価されているが、一部のリスクまたはリソース見積もりが不十分 |
| 10〜13 | 実現可能性の評価が表面的で、現実的な課題への言及が少ない |
| 0〜9  | 実現可能性の検討がほぼなく、アイデアの実行可能性が不明 |

---

### 4. ペルソナ明確性 【配点：20点】

> **目的：** ターゲット顧客のペルソナが具体的に定義されており、ビジネス全体がそのペルソナの課題解決に集中しているかを確認する。

- [ ] **具体的なペルソナ**（年齢・職業・ライフスタイル・悩みなど）が設定されているか
- [ ] ペルソナが抱える**課題・ペインポイント**が具体的に描写されているか
- [ ] このビジネスがペルソナの課題を**どのように解決するか**が明確に示されているか
- [ ] ペルソナが「**なぜこのサービスを選ぶか**」の動機が説明されているか
- [ ] ペルソナに対する**獲得チャネル**（どこで・どうやって出会うか）が示されているか
- [ ] ペルソナが複数いる場合、**プライマリ（主要）ペルソナ**が特定されているか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | ペルソナが具体的で、課題・解決策・獲得チャネルまで一貫して定義されている |
| 14〜17 | おおむね明確だが、ペルソナの一部（動機・獲得方法等）が不十分 |
| 10〜13 | ペルソナが抽象的で、「誰のためか」が漠然としている |
| 0〜9  | ターゲット顧客の定義がほぼなく、誰向けのビジネスかが不明 |

---

### 5. 展開計画の具体性 【配点：20点】

> **目的：** ビジネスアイデアを実際に動かすための段階的な展開計画が具体的に示されているかを確認する。

- [ ] **フェーズ分け**（例：検証フェーズ→MVP→スケール）が定義されているか
- [ ] **最初の100日間（または3ヶ月）で何をするか**が具体的に示されているか
- [ ] **成功指標（KPI）**が定量的に設定されているか（顧客数・売上・継続率等）
- [ ] 最初の顧客を**どうやって獲得するか**の具体的な方法が記載されているか
- [ ] 事業を**スケールさせるための仕組み**（口コミ・SEO・パートナーシップ等）が示されているか
- [ ] 長期的なビジョン（3〜5年後のあるべき姿）が示されているか

**採点目安**

| 点数 | 基準 |
|------|------|
| 18〜20 | フェーズ・KPI・獲得方法・スケール戦略が揃い、長期ビジョンまで示されている |
| 14〜17 | おおむね具体的だが、一部の計画（KPIや獲得方法等）が不十分 |
| 10〜13 | 展開計画が抽象的で、「まず何をするか」が不明確 |
| 0〜9  | 展開計画がほぼなく、アイデアのままで止まっている |

---

## 採点方法

### 総合点：100点満点

| 評価 | 点数 | 判定 |
|------|------|------|
| S | 90〜100点 | 事業計画書・提案書作成ステップへ進む準備完了 |
| A | 80〜89点 | 軽微な補強後に次ステップへ進める |
| B | 70〜79点 | 修正推奨（特定観点の深掘りが必要） |
| C | 60〜69点 | 大幅な再検討が必要 |
| D | 59点以下 | アイデアの根本的な見直しが必要 |

## 重要な注意事項

> **レビュワーは元ファイルを絶対に編集してはならない**
>
> レビューはあくまで「フィードバックの提供」であり、修正作業は作成者自身が行う。
> 改善点は具体的に指摘し、修正案は「提案」として記載すること。

---

## 使用方法

1. このプロンプトと一緒に、レビュー対象のビジネスアイデア文書を貼り付ける
2. 「このビジネスアイデアをレビューしてください」と依頼する
3. S/A判定であれば `docs/business-proposal.md` を使って事業計画書に発展させることを推奨する
4. B以下の場合は改善提案を反映してから再レビューを実施する
